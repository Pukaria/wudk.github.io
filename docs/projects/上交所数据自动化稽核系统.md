# 上交所数据自动化稽核系统

## 📌 项目背景

在日常金融数据运营中，需要定期从上海证券交易所官网抓取市场数据，与内部数据库进行比对，以验证数据的一致性和完整性。这项工作涉及**指数行情、融资融券、股票成交、基金成交、债券市场**等多个数据源，手动操作耗时 2-3 小时，且容易因网页改版或数据格式变化导致失败。

本系统实现了一个**全自动化的数据稽核流水线**，从数据抓取到最终比对报告生成，全程无人值守，并支持**钉钉机器人实时通知**，将每日稽核时间从 2-3 小时缩短至 **5 分钟以内**。

## 📥 阶段一：数据抓取（8 个数据源）

|:---:|:---|:---|:---|
| 1 | 上交所实时行情接口 | HTTP GET + JSON 解析 | 主要指数（上证综指、上证50、科创50等）的收盘价、涨跌幅、成交额 |
| 2 | 中证指数有限公司 | HTTP GET + JSON 解析 | 中证1000 指数昨收 |

| 5 | 上交所基金成交概况 | Selenium 动态渲染 | 基金挂牌数、ETF/REITs/LOF 成交量与成交额 |
| 6 | 上交所债券市场概况 | JSONP API | 债券只数、面值、市值（合计） |

### 关键技术亮点

**1. 交易日自适应计算**

使用 `chinese_calendar` 库自动识别工作日，确保程序仅在交易日执行抓取：

```python
def get_last_trading_day(start_date=None):
    current = start_date or datetime.now().date()
    for _ in range(7):
        current -= timedelta(days=1)
        if is_workday(current):
            return current.strftime('%Y-%m-%d')
    return start_date.strftime('%Y-%m-%d')
```

**2. 债券 JSONP 接口解析**

上交所债券接口返回的是 JSONP 格式（`jsonpCallback({...})`），系统使用正则表达式提取并解析：

```python
raw = resp.text
match = re.search(r'jsonpCallback\((.*)\)', raw)
if match:
    json_str = match.group(1)
    return json.loads(json_str)
```

## 🔄 阶段二：模板映射（含派生指标计算）

抓取的原始数据与业务模板之间存在**字段名称差异**，需要通过映射规则进行转换。

### 映射方式

| 映射类型 | 说明 | 示例 |
|:---|:---|:---|
| **手动映射表** | 预定义 `模板指标名 → 原始指标名` 的对应关系 | `'上证综合指数:收盘价:日'` → `'上证指数指数'` |
| **精确匹配** | 模板名称与原始名称完全一致 | 直接取用 |
| **核心词匹配** | 去除后缀（`:收盘价:日`、`:上海证券交易所:日`）后匹配 | `'股票挂牌数:上海证券交易所:日'` → `'股票挂牌数'` |
| **模糊匹配** | 双向包含关系匹配 | 用于处理细微名称差异 |
| **派生计算** | 基于多个原始指标计算得出 | 详见下方 |

### 派生指标计算

以下两个指标无法直接抓取，需通过公式计算：

**融资偿还额**：
```
融资偿还额 = 上期融资余额 - 本期融资余额 + 本期融资买入额
```

**融券偿还量**：
```
融券偿还量 = 上期融券余量 - 本期融券余量 + 本期融券卖出量
```

系统自动从原始数据中查找最近两期的融资/融券数据，按公式计算并填充。

## 📊 阶段三：数据库比对

### 数据源

| 数据源 | 说明 |
|:---|:---|
| **映射结果 Excel** | 阶段二生成的映射后数据 |
| **SQL Server（生产库）** | 包含 `usrXHGSJBHG`、`usrXHGSJBHGMY`、`usrXHGSJBHYCW`、`usrXHGSJBQY` 四个分表 |

### 比对逻辑

1. **按 ZBDM（指标代码）关联**，取数据库中该指标**最新一条记录**（按 JZRQ 降序）
2. 使用 **CTE（公共表表达式）+ 窗口函数** 进行高效查询
3. 逐指标比对数值，计算**绝对差异**和**相对差异**

```python
sql = """
WITH AllData AS (
    SELECT XXFBRQ, ZBDM, ZBSJ, JZRQ, XGRY, XGSJ FROM usrXHGSJBHG
    UNION ALL
    SELECT XXFBRQ, ZBDM, ZBSJ, JZRQ, XGRY, XGSJ FROM usrXHGSJBHGMY
    -- ... 更多分表
),
Ranked AS (
    SELECT *, ROW_NUMBER() OVER (PARTITION BY ZBDM ORDER BY JZRQ DESC) AS rn
    FROM AllData WHERE ZBDM IN ({placeholders})
)
SELECT ZBDM, JZRQ, ZBSJ, XXFBRQ, XGSJ FROM Ranked WHERE rn = 1
"""
```

### 比对状态分类

| 比对状态 | 判定条件 | 业务含义 |
|:---|:---|:---|
| **完全一致** | 绝对差异 = 0 | 数据完全匹配 ✅ |
| **基本一致** | 相对差异 < 0.01% | 差异微小，可接受 ✅ |
| **存在偏差** | 相对差异 0.01% ~ 1% | 需人工核对 ⚠️ |
| **差异较大** | 相对差异 ≥ 1% | 需重点核查 🔴 |
| **数据库无数据** | 数据库无该指标记录 | 需新增录入 🔴 |

## 📱 阶段四：钉钉智能通知

系统集成钉钉机器人，实现**执行结果实时推送**和**异常告警**。

### 通知机制

| 通知场景 | 触发条件 | 接收人 |
|:---|:---|:---|
| 每日正常报告 | 执行完成，无异常 | 群内@相关人员 |
| 问题告警 | 存在“差异较大”或“数据库无数据” | @指定人员（18039726681） |
| 程序异常 | 任何阶段发生异常 | @指定人员 |

### 安全模式

使用**加签（HMAC-SHA256）**模式，确保 Webhook 请求的安全性：

```python
def _generate_sign(self):
    timestamp = str(round(time.time() * 1000))
    secret_enc = self.secret.encode('utf-8')
    string_to_sign = f'{timestamp}\n{self.secret}'
    hmac_code = hmac.new(secret_enc, string_to_sign_enc, digestmod=hashlib.sha256).digest()
    sign = base64.b64encode(hmac_code).decode('utf-8')
    return timestamp, sign
```
