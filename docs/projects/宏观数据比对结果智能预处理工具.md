# 宏观数据比对结果智能预处理工具

## 📌 项目背景

在上一阶段完成两期数据的自动比对后（[宏观数据比对工具](宏观数据自动比对工具.md)），生成了一个包含差异标记的 Excel 文件。然而，原始的比对结果（`JZRQ_same = True/False`）对于业务人员来说不够直观，无法直接判断每条记录应该如何处理。

本工具作为**比对工具的配套模块**，实现了**自动化的备注填充与分级标注**，将技术指标转化为业务可读的审核结论，大幅减少了人工逐条判断的工作量。

| Datetime | 日期时间计算与稽核 |

## 🧠 核心逻辑解读

工具的核心任务是**基于多条优先级规则**，为比对结果中的每条记录自动填充“备注”列，实现智能分级标注。

决策流程如下：

```mermaid
flowchart TD
    A[读取比对结果Excel] --> B{备注是否已为'正确'?}
    B -->|是| C[跳过该行]
    B -->|否| D{OPERATORNAME 是否为'无(unknown)'?}
    D -->|是| E[标注: 无需审核]
    D -->|否| F{JZRQ_same 是否为 False?}
    F -->|是| G[标注: 需要人工审核]
    F -->|否| H[获取披露频率 & 最后修改时间]
    H --> I{频率 = 999?}
    I -->|是| J[标注: 需人工审核]
    I -->|否| K[计算距今天数]
    K --> L{距今天数 > 92天?}
    L -->|是| M[标注: 超出周期需要审核]
    L -->|否| N{距今天数 ≤ 披露频率?}
    N -->|是| O[标注: 时间之内正确]
    N -->|否| P{距今天数 > 频率 × 3?}
    P -->|是| Q[标注: 疑似停更]
    P -->|否| R[标注: 超出周期需要审核]
```

### 关键规则说明

| 优先级 | 判断条件 | 标注结果 | 业务含义 |
|:---:|:---|:---|:---|
| 1 | `OPERATORNAME` = "无(unknown)" | **无需审核** | 该指标尚未分配负责人，暂不纳入审核流程 |
| 2 | `JZRQ_same` = False | **需要人工审核** | 两期数据的关键日期不一致，需人工介入确认 |
| 3 | 披露频率 = 999 | **需人工审核** | 特殊标记，需人工判断 |
| 4 | 距今天数 > 92天（约1个季度） | **超出周期需要审核** | 数据超过一个季度未更新，需重新稽核 |
| 5 | 距今天数 ≤ 披露频率 | **时间之内正确** | 数据在预期更新周期内，状态正常 |
| 6 | 距今天数 > 披露频率 × 3 | **疑似停更** | 数据已超过预期更新周期的3倍未刷新，可能已停止更新 |
| 7 | 其他情况（频率 < 距今天数 ≤ 频率×3） | **超出周期需要审核** | 数据已超出正常更新周期，需确认 |
