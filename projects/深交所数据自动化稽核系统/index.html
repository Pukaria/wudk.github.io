深交所数据自动化稽核系统

📌 项目背景

在完成上交所数据自动化后，深交所数据的稽核工作仍依赖人工每日登录官网、手动导出Excel、逐项比对，效率低下且易出错。同时，深交所数据源具有多接口、多格式、多频次的特点（JSON API、HTML表格、实时行情接口等），对自动化系统的健壮性和兼容性提出了更高要求。

本系统作为双交易所自动化体系的深交所侧实现，实现了从数据抓取 → 智能映射 → 数据库比对 → 钉钉通知的全链路自动化，与上交所系统协同工作，共同构成完整的市场数据质量保障体系。

📥 阶段一：数据抓取（8 个数据源）

系统自动从以下 8 个数据源并行/串行抓取数据：

序号 数据源 接口类型 关键数据
1 市场总貌 HTTP JSON 上市公司数、上市证券数、总市值、流通市值、成交金额（含各板块细分）
2 日度成交量 HTTP JSON 主板/创业板/深市合计的成交量
3 基金日度概况 HTTP JSON ETF/LOF/公募REITs 成交量、成交金额
4 股票指标 HTTP JSON（3 个 Tab） 主板/创业板/深圳市场的：平均市盈率、平均换手率、总股本、流通股本等
5 指数行情 HTTP JSON（按日期范围） 近一周各指数的每日收盘价
6 融资融券 HTTP JSON 融资余额、融券余额、融资买入额、融券卖出量、融券余量
7 深港通汇率 HTTP JSON（分页循环） 参考汇率买入价、参考汇率卖出价
8 实时指数 实时行情接口 深证成指、深证综指、创业板指的实时价格

关键技术亮点

1. 多日期范围抓取

不同于上交所的单日抓取，深交所支持按日期范围批量抓取历史数据：

def get_trading_dates(start_date: str, end_date: str) -> list:
    """获取指定日期范围内的所有交易日"""
    start = datetime.strptime(start_date, "%Y-%m-%d")
    end = datetime.strptime(end_date, "%Y-%m-%d")
    dates = []
    current = start
    while current <= end:
        if is_workday(current):
            dates.append(current.strftime("%Y-%m-%d"))
        current += timedelta(days=1)
    return dates

2. 健壮的 Session 管理 + 重试机制

使用 urllib3.Retry 实现自动重试，对 5xx 错误和超时进行容错：

def create_session():
    session = requests.Session()
    retry = Retry(total=3, backoff_factor=1, status_forcelist=[500, 502, 503, 504])
    adapter = HTTPAdapter(max_retries=retry)
    session.mount("http://", adapter)
    session.mount("https://", adapter)
    return session

🔄 阶段二：智能映射（含派生指标计算）

映射方式（4 层匹配策略）

层级 匹配方式 说明 示例
1 精确匹配 模板名称与爬虫名称完全一致 融资余额 → 融资余额

4 模糊匹配 双向包含关系 农林牧渔指数 ↔ 农林指数

系统支持从数据库实时计算派生指标，当前配置了 7 个衍生求和指标：

📊 阶段三：数据库比对（多日期维度）

比对逻辑

与上交所版本不同，深交所系统支持按日期维度进行全量比对：

  1. 按 ZBDM + JZRQ 联合匹配，查询数据库中该指标在指定日期的最新记录
  2. 使用 CTE + 窗口函数 高效查询（ROW_NUMBER() OVER (PARTITION BY ZBDM, JZRQ ORDER BY XGSJ DESC)）
  3. 逐行计算绝对差异和相对差异
sql = """
WITH AllData AS (
    SELECT ZBDM, JZRQ, ZBSJ, XGSJ FROM usrXHGSJBHG
    UNION ALL ...
),
Ranked AS (
    SELECT ZBDM, JZRQ, ZBSJ,
        ROW_NUMBER() OVER (PARTITION BY ZBDM, JZRQ ORDER BY XGSJ DESC) AS rn
    FROM AllData
    WHERE ZBDM IN ({placeholders}) AND JZRQ IN ({date_placeholders})
)
SELECT ZBDM, JZRQ, ZBSJ FROM Ranked WHERE rn = 1
"""

比对状态分类

比对状态 判定条件 业务含义
完全一致 绝对差异 = 0 数据完全匹配 ✅
基本一致 相对差异 < 0.01% 差异微小，可接受 ✅
存在偏差 相对差异 0.01% ~ 1% 需人工核对 ⚠️
差异较大 相对差异 ≥ 1% 需重点核查 🔴
数据库无数据 数据库无该指标+日期记录 需新增录入 🔴

📱 阶段四：钉钉智能通知（含 LRMBMX 标识）

业务标识

系统在钉钉通知中引入 LRMBMX（新宏观-D1-深圳证券交易所）作为关键业务标识，便于群内人员快速识别消息所属业务线：

LRMBMX = "新宏观-D1-深圳证券交易所"

通知内容结构

信息模块 内容
关键标识 LRMBMX（业务线标识）
数据时效 爬虫最新日期 vs 数据库最新日期
比对统计 完全一致/基本一致/存在偏差/数据库无数据的数量和占比
偏差示例 存在偏差时，展示前5条具体差异（含指标名、日期、数值对比）
缺失示例 数据库无数据时，展示前5条缺失项
生成文件 比对结果 Excel 文件路径
